# 数据接收功能测试指南

## 一、准备工作

### 1. 启动后端服务

```bash
cd backend

venv\Scripts\activate
```

```bash
cd forntend
npm run dev
```

### 3. 确认数据库连接

确保数据库中有至少一个设备记录（device表），用于测试。

## 二、前端界面测试

### 步骤1：打开设备数据日志页面

1. 在浏览器中访问前端应用（通常是 `http://localhost:5173`）
2. 导航到"设备数据日志"页面

### 步骤2：启动数据接收服务

1. 点击页面顶部的"数据接收"按钮
2. 在弹出的模态框中：
   - 设置"测试设备ID"（确保该设备ID在数据库中存在）
   - 可选择"自动刷新数据列表"选项
   - 点击"开始接收"按钮

### 步骤3：观察接收状态

- **状态显示**：应显示"运行中"（绿色）
- **接收总数**：每3秒自动增加
- **最后接收时间**：显示最新接收时间
- **错误次数**：如果有错误会显示

### 步骤4：验证数据

1. 点击"关闭"按钮关闭模态框
2. 在数据列表中查看新接收的数据
3. 如果启用了"自动刷新数据列表"，数据会自动更新

### 步骤5：测试手动发送

1. 重新打开数据接收模态框
2. 点击"停止接收"停止自动接收
3. 点击"手动发送测试数据"按钮
4. 观察是否成功接收

### 步骤6：测试跌倒事件自动创建

1. 在数据接收模态框中，修改测试数据生成逻辑（或等待随机生成跌倒数据）
2. 当生成 `is_fall_confirmed = true` 的数据时
3. 检查"事件管理"页面，应该看到新创建的待处理事件

### 方法2：使用 Python 脚本

创建测试脚本 `test_data_reception.py`:

```python
import requests
import json
import time
from datetime import datetime

API_BASE_URL = "http://localhost:8000/api/v1/data-reception"

def send_test_data(device_id=1, is_fall=False):
    """发送测试数据"""
    now = int(time.time())
    
    data = {
        "device_id": device_id,
        "timestamp": now,
        "relative_time": now,
        "accelerometer": {
            "x": 2.456 if is_fall else 0.123,
            "y": -1.234 if is_fall else -0.046,
            "z": 9.876 if is_fall else 1.023
        },
        "gyroscope": {
            "x": 15.234 if is_fall else 1.254,
            "y": -8.765 if is_fall else -0.754,
            "z": 12.345 if is_fall else 0.325
        },
        "location": {
            "x": 2.35,
            "y": 3.67,
            "z": 0.5 if is_fall else 0,
            "accuracy": 1.25,
            "position_quality": "high" if is_fall else "medium"
        },
        "fall_detection": {
            "state": 4 if is_fall else 0,
            "state_description": "确认跌倒" if is_fall else "正常",
            "confidence": 0.95 if is_fall else 0.15,
            "is_fall_confirmed": is_fall,
            "impact_force": 6.80 if is_fall else 0.0,
            "direction": "前" if is_fall else "",
            "fall_time": now if is_fall else 0
        },
        "system_status": {
            "wifi_connected": True,
            "server_connected": True,
            "battery_level": 85
        }
    }
    
    try:
        response = requests.post(
            f"{API_BASE_URL}/receive",
            json=data,
            headers={"Content-Type": "application/json"}
        )
        response.raise_for_status()
        result = response.json()
        print(f"✅ 数据接收成功: {result}")
        return result
    except requests.exceptions.RequestException as e:
        print(f"❌ 数据接收失败: {e}")
        if hasattr(e, 'response') and e.response is not None:
            print(f"   错误详情: {e.response.text}")
        return None

def check_status():
    """检查服务状态"""
    try:
        response = requests.get(f"{API_BASE_URL}/status")
        response.raise_for_status()
        result = response.json()
        print(f"📊 服务状态: {result}")
        return result
    except requests.exceptions.RequestException as e:
        print(f"❌ 获取状态失败: {e}")
        return None

if __name__ == "__main__":
    print("=" * 60)
    print("数据接收功能测试")
    print("=" * 60)
    
    # 1. 检查服务状态
    print("\n1. 检查服务状态...")
    check_status()
    
    # 2. 发送正常数据
    print("\n2. 发送正常数据...")
    send_test_data(device_id=1, is_fall=False)
    
    time.sleep(1)
    
    # 3. 发送跌倒数据（会自动创建事件）
    print("\n3. 发送跌倒数据（会自动创建事件）...")
    send_test_data(device_id=1, is_fall=True)
    
    print("\n✅ 测试完成！")
    print("\n请检查：")
    print("1. 数据库 device_data_log 表中是否有新数据")
    print("2. 如果发送了跌倒数据，event 表中是否有新事件")
```

```sql
-- 查看最新接收的数据
SELECT * FROM device_data_log

       COUNT(CASE WHEN is_fall_confirmed = 1 THEN 1 END) as fall_count

```

### 2. 检查 event 表（跌倒事件）

```sql
-- 查看自动创建的跌倒事件
SELECT * FROM event 
WHERE event_type = 'fall' 
  AND event_status = 'unhandled'
ORDER BY event_timestamp DESC 
LIMIT 10;

-- 查看事件参数（包含日志ID）
SELECT 
    event_id,
    event_type,
    related_user_id,
    trigger_device_id,
    event_timestamp,
    event_status,
    JSON_EXTRACT(event_params, '$.log_id') as log_id,
    JSON_EXTRACT(event_params, '$.fall_confidence') as fall_confidence
FROM event 
WHERE event_type = 'fall'
ORDER BY event_timestamp DESC;
```

### 3. 验证数据关联

```sql
-- 查看设备数据日志及其关联的事件
SELECT 
    ddl.id as log_id,
    ddl.device_id,
    ddl.is_fall_confirmed,
    ddl.fall_state_description,
    e.event_id,
    e.event_status,
    e.event_timestamp
FROM device_data_log ddl
LEFT JOIN event e ON JSON_EXTRACT(e.event_params, '$.log_id') = ddl.id
WHERE ddl.is_fall_confirmed = 1
ORDER BY ddl.created_at DESC
LIMIT 10;
```

## 五、常见问题排查

### 问题1：数据接收失败，返回400错误

**可能原因**：
- 数据格式不正确
- 缺少必需字段
- 设备ID不存在

**解决方法**：
- 检查请求的JSON格式是否正确
- 确认所有必需字段都已提供
- 验证设备ID在数据库中存在

### 问题2：数据接收成功但未创建事件

**可能原因**：
- `is_fall_confirmed` 为 false
- 设备未关联到用户（`device.elderly_user_id` 为 NULL）

**解决方法**：
- 确认发送的数据中 `is_fall_confirmed = true`
- 检查设备表中该设备是否有关联的用户ID

### 问题3：前端显示"数据接收失败"

**可能原因**：
- 后端服务未启动
- API地址配置错误
- CORS问题

**解决方法**：
- 确认后端服务正在运行（访问 `http://localhost:8000/docs`）
- 检查浏览器控制台的错误信息
- 确认前端API配置正确

### 问题4：前端自动接收不工作

**可能原因**：
- JavaScript定时器被清除
- 页面刷新导致状态丢失

**解决方法**：
- 重新打开数据接收模态框
- 点击"开始接收"重新启动

## 七、性能测试

### 批量发送测试

```python
import requests
import time
from concurrent.futures import ThreadPoolExecutor

def send_data_batch(count=100):
    """批量发送数据"""
    success = 0
    failed = 0
    
    def send_one(i):
        nonlocal success, failed
        data = {
            "device_id": 1,
            "timestamp": int(time.time()) + i,
            "relative_time": int(time.time()) + i,
            "accelerometer": {"x": 0.123, "y": -0.046, "z": 1.023},
            "gyroscope": {"x": 1.254, "y": -0.754, "z": 0.325},
            "location": {"x": 2.35, "y": 3.67, "z": 0, "accuracy": 1.25, "position_quality": "medium"},
            "fall_detection": {"state": 0, "state_description": "正常", "confidence": 0.15, "is_fall_confirmed": False, "impact_force": 0.0, "direction": "", "fall_time": 0},
            "system_status": {"wifi_connected": True, "server_connected": True, "battery_level": 85}
        }
        try:
            response = requests.post("http://localhost:8000/api/v1/data-reception/receive", json=data, timeout=5)
            if response.status_code == 200:
                success += 1
            else:
                failed += 1
        except:
            failed += 1
    
    start_time = time.time()
    with ThreadPoolExecutor(max_workers=10) as executor:
        executor.map(send_one, range(count))
    end_time = time.time()
    
    print(f"发送 {count} 条数据:")
    print(f"  成功: {success}")
    print(f"  失败: {failed}")
    print(f"  耗时: {end_time - start_time:.2f} 秒")
    print(f"  平均: {(end_time - start_time) / count * 1000:.2f} 毫秒/条")

if __name__ == "__main__":
    send_data_batch(100)
```
