# 数据接收功能使用说明

## 功能概述

数据接收功能用于接收外部IMU设备（如智能手表）通过WiFi发送的传感器数据，并自动保存到数据库。如果检测到确认跌倒，系统会自动创建待处理事件。

## 后端接口

### 1. 接收数据接口

**端点**: `POST /api/v1/data-reception/receive`

**请求格式**: JSON

**请求示例**:

**响应示例**:

## 外部设备配置

### 设备端代码示例（Python）

```python
import requests
import json
import time
from datetime import datetime

# 服务器地址（根据实际情况修改）
SERVER_URL = "http://your-server-ip:8000/api/v1/data-reception/receive"

def send_imu_data(device_id, accel_data, gyro_data, location_data, 
                  fall_detection, system_status):
    """发送IMU数据到服务器"""
    
    data = {
        "device_id": device_id,
        "timestamp": int(time.time()),
        "relative_time": int(time.time()),
        "accelerometer": {
            "x": accel_data["x"],
            "y": accel_data["y"],
            "z": accel_data["z"]
        },
        "gyroscope": {
            "x": gyro_data["x"],
            "y": gyro_data["y"],
            "z": gyro_data["z"]
        },
        "location": {
            "x": location_data["x"],
            "y": location_data["y"],
            "z": location_data["z"],
            "accuracy": location_data.get("accuracy", 999.0),
            "position_quality": location_data.get("position_quality", "unknown")
        },
        "fall_detection": {
            "state": fall_detection.get("state", 0),
            "state_description": fall_detection.get("state_description", "正常"),
            "confidence": fall_detection.get("confidence", 0.0),
            "is_fall_confirmed": fall_detection.get("is_fall_confirmed", False),
            "impact_force": fall_detection.get("impact_force", 0.0),
            "direction": fall_detection.get("direction", ""),
            "fall_time": fall_detection.get("fall_time", 0)
        },
        "system_status": {
            "wifi_connected": system_status.get("wifi_connected", False),
            "server_connected": system_status.get("server_connected", False),
            "battery_level": system_status.get("battery_level", 0)
        }
    }
    
    try:
        response = requests.post(
            SERVER_URL,
            json=data,
            headers={"Content-Type": "application/json"},
            timeout=5
        )
        response.raise_for_status()
        result = response.json()
        print(f"✅ 数据发送成功: {result}")
        return result
    except requests.exceptions.RequestException as e:
        print(f"❌ 数据发送失败: {e}")
        return None

# 使用示例
if __name__ == "__main__":
    # 模拟数据

```

### 设备端代码示例（Arduino/ESP32）

```cpp
#include <WiFi.h>
#include <HTTPClient.h>
#include <ArduinoJson.h>

const char* ssid = "your-wifi-ssid";
const char* password = "your-wifi-password";
const char* serverUrl = "http://your-server-ip:8000/api/v1/data-reception/receive";

void sendIMUData(int deviceId, float accelX, float accelY, float accelZ,
                  float gyroX, float gyroY, float gyroZ,
                  float locX, float locY, float locZ,
                  bool isFallConfirmed) {
    if (WiFi.status() != WL_CONNECTED) {
        Serial.println("WiFi未连接");
        return;
    }
    
    HTTPClient http;
    http.begin(serverUrl);
    http.addHeader("Content-Type", "application/json");
    
    // 构建JSON数据
    DynamicJsonDocument doc(1024);
    doc["device_id"] = deviceId;
    doc["timestamp"] = millis() / 1000;
    doc["relative_time"] = millis() / 1000;
    
    doc["accelerometer"]["x"] = accelX;
    doc["accelerometer"]["y"] = accelY;
    doc["accelerometer"]["z"] = accelZ;
    
    doc["gyroscope"]["x"] = gyroX;
    doc["gyroscope"]["y"] = gyroY;
    doc["gyroscope"]["z"] = gyroZ;
    
    doc["location"]["x"] = locX;
    doc["location"]["y"] = locY;
    doc["location"]["z"] = locZ;
    doc["location"]["accuracy"] = 1.25;
    doc["location"]["position_quality"] = "medium";
    
    doc["fall_detection"]["state"] = isFallConfirmed ? 4 : 0;
    doc["fall_detection"]["state_description"] = isFallConfirmed ? "确认跌倒" : "正常";
    doc["fall_detection"]["confidence"] = isFallConfirmed ? 0.95 : 0.15;
    doc["fall_detection"]["is_fall_confirmed"] = isFallConfirmed;
    doc["fall_detection"]["impact_force"] = isFallConfirmed ? 6.80 : 0.0;
    doc["fall_detection"]["direction"] = isFallConfirmed ? "前" : "";
    doc["fall_detection"]["fall_time"] = isFallConfirmed ? millis() / 1000 : 0;
    
    doc["system_status"]["wifi_connected"] = true;
    doc["system_status"]["server_connected"] = true;
    doc["system_status"]["battery_level"] = 85;
    
    String jsonString;
    serializeJson(doc, jsonString);
    
    int httpResponseCode = http.POST(jsonString);
    
    if (httpResponseCode > 0) {
        String response = http.getString();
        Serial.println("✅ 数据发送成功: " + response);
    } else {
        Serial.println("❌ 数据发送失败: " + String(httpResponseCode));
    }
    
    http.end();
}
```

## 前端监控功能

### 使用步骤

1. **打开数据接收页面**
   - 进入"设备数据日志"页面
   - 点击页面顶部的"数据接收"按钮

2. **查看接收状态**
   - 模态框会显示当前的接收统计信息：
     - 状态：运行中/已停止
     - 接收总数：已接收的数据条数
     - 最后接收时间：最后一次接收数据的时间
     - 错误次数：接收失败的次数

3. **开始监控**
   - 点击"开始监控"按钮
   - 系统会每2秒自动刷新接收统计信息
   - 如果启用了"自动刷新数据列表"，数据列表也会自动更新

4. **配置选项**
   - **自动刷新数据列表**：勾选后，监控模式下会自动刷新数据列表

## 数据流程

```
外部IMU设备
    ↓ (WiFi POST请求)
FastAPI后端 (/api/v1/data-reception/receive)
    ↓
1. 添加服务器接收时间戳
2. 控制台显示数据详情
3. 转换数据格式
4. 保存到 device_data_log 表
5. 如果 is_fall_confirmed = true，自动创建事件
6. 更新接收统计
    ↓
返回成功响应给设备
```

## 注意事项

1. **设备ID验证**：确保发送的 `device_id` 在数据库的 `device` 表中存在
2. **用户关联**：如果希望自动创建事件，设备必须关联到用户（`device.elderly_user_id` 不为 NULL）
3. **网络配置**：确保设备能够访问后端服务器的IP地址和端口
4. **数据格式**：严格按照JSON格式发送数据，所有必需字段都必须提供
5. **时间戳**：`timestamp` 和 `relative_time` 应为Unix时间戳（秒）
6. **跌倒检测**：只有当 `is_fall_confirmed = true` 时才会自动创建事件

## 故障排查

### 问题1：数据接收失败（400错误）

**可能原因**：
- 数据格式不正确
- 缺少必需字段
- 数据类型不匹配

**解决方法**：
- 检查JSON格式是否正确
- 确认所有必需字段都已提供
- 验证数据类型（如 `device_id` 应为整数）

### 问题2：数据接收成功但未创建事件

**可能原因**：
- `is_fall_confirmed` 为 false
- 设备未关联到用户

**解决方法**：
- 确认发送的数据中 `is_fall_confirmed = true`
- 检查设备表中该设备是否有关联的用户ID

### 问题3：前端统计不更新

**可能原因**：
- 未点击"开始监控"
- 网络连接问题
- 后端服务未运行

**解决方法**：
- 点击"开始监控"按钮
- 检查浏览器控制台的错误信息
- 确认后端服务正在运行
